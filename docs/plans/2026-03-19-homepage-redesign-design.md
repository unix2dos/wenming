# Homepage Redesign Design

**Date:** 2026-03-19
**Scope:** `/` 首页信息架构与视觉密度重构
**Goal:** 把首页从“说明页”收敛成“测试启动页”，让用户进入后优先点击 `开始测试`。

## Problem

当前首页存在三类结构性问题：

1. 首屏目标不唯一
   当前首屏同时承担“品牌介绍、测试入口、直接起名入口、结果预览前置信号”，导致主 CTA 没有绝对优势。

2. 标题重量过高
   长结果型标题占用过多垂直空间，用户进入页面后先感知到“内容很多”，而不是“动作很清楚”。

3. 首页重复解释
   “为什么先测”“适合谁来测”“不走玄学/先测方向/再去起名”等内容表达重复，拉高了页面密度，但没有成比例提升理解效率。

## Design Principles

1. 首屏只服务一个动作
   首页首屏唯一主任务是点击 `开始测试`。

2. 解释下沉，启动前置
   结果预览和业务次入口保留，但全部下沉到首屏以下。

3. 视觉重量服从转化顺序
   品牌、动作型标题、副文案、主 CTA 形成单条阅读链，不再让标签、卡片、双按钮分流注意力。

4. 次入口保留但降权
   `已有名字？看名字好不好` 只保留在顶部右侧；`直接起名` 移到第二屏做轻入口。

## Approaches

### Option A: 极简启动页

- 首屏只保留品牌、动作型标题、副文案、主 CTA、顶部弱链接
- 第二屏保留命名原型预览
- 第三块只保留轻量 `直接起名` 入口

**Pros**
- 转化路径最清楚
- 最符合“先测试后理解”的目标
- 最容易解决“标题太大、信息太多”

**Cons**
- 首页的解释力会更依赖第二屏

**Decision**
- 采用。用户已确认。

### Option B: 轻信息首页

- 首屏保留主 CTA，同时保留 `直接起名`
- 第二屏展示预览与解释

**Pros**
- 兼顾两条路径

**Cons**
- 重新回到双主任务竞争

### Option C: 导航型首页

- 首屏展示测试、起名、已有名字评分三条路径

**Pros**
- 业务覆盖完整

**Cons**
- 和当前问题同源，只是更整齐

## Approved Structure

### Hero

保留元素：

- 品牌 `问名`
- 顶部弱链接 `已有名字？看名字好不好`
- 动作型主标题
- 一句副文案
- 主按钮 `开始测试`

移除元素：

- `直接起名` 按钮
- 标签组 `不走玄学 / 先测方向 / 再去起名`
- 原型预览卡
- `为什么先测`
- `适合谁来测`

### Below the Fold

首页首屏以下只保留两块：

1. `命名原型预览`
   作用是降低测试结果不确定性，但不抢首屏注意力。

2. `直接起名` 轻入口
   形式是一行说明 + 一个次级按钮，不做大卡片。

## Wireframe

```text
+-----------------------------------------------------------+
| 问名                                      已有名字？看看  |
|                                                           |
| 先测方向，再起名                                          |
| 用 30 秒先找到你们更合拍的命名方向。                      |
|                                                           |
| [ 开始测试 ]                                              |
+-----------------------------------------------------------+

+-----------------------------------------------------------+
| 三个命名原型预览                                           |
| [雅正型] [空明型] [天真型]                                 |
+-----------------------------------------------------------+

+-----------------------------------------------------------+
| 已经有明确方向了？                                         |
| 也可以直接输入姓氏与偏好开始起名。                         |
| [ 直接起名 ]                                              |
+-----------------------------------------------------------+
```

## Copy Rules

1. 主标题必须是动作型短句
   不再使用“测你家起名更偏大雅还是大俗”这类结果型长标题作为首页主标题。

2. 副文案只说一件事
   只保留“先测方向再起名”的一句白话解释，不重复展开。

3. 第二屏只做结果预览，不做长教育
   每张卡只保留阵营、类型、短解释、示例名。

4. `直接起名` 只作为次路径出现
   它必须可见，但不能和首页主 CTA 形成平权。

## Visual Direction

基于现有站点风格，首页改成更轻的 editorial minimal：

- 减少首屏纵向模块数量
- 压缩标题字号和最大宽度
- 增加标题与 CTA 之间的负空间
- 第二屏卡片整体变轻，不做过重说明块

设计系统依据：

- Single CTA focus
- Large but controlled typography
- Below-the-fold proof, not hero proof

## Acceptance Mapping

完成后首页应满足：

1. 用户进入首页后，第一眼是 `开始测试`
2. 首屏不再显得“标题太大、信息太多”
3. 原型预览保留，但必须在首屏以下
4. `直接起名` 仍可达，但明显是次级路径
