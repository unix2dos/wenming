# 「好名字」产品设计方案

> 为准父母打造的"去玄学化、重审美"中文宝宝起名 + 名字打分 Web 产品

---

## Phase 0 · 问题定义

> **一句话定义：** 以"大雅大俗 + 民国风骨"为核心审美标准，为准父母提供 AI 驱动的起名推荐与名字评分服务，与市面上重五行八字的竞品形成差异化。

| 维度 | 定义 |
|---|---|
| **用户** | 准父母（25-35 岁），怀孕中后期或宝宝已出生 |
| **场景** | ① **主力·决策型**：输入姓氏+偏好 → 获取推荐 ② **辅助·探索型**：输入已有名字 → 获得打分 |
| **成功标准** | ① 用户获得 ≥5 个满意候选名 ② 打分让用户觉得"说到点上了" ③ 免费→付费转化 |
| **差异化** | 拒绝生僻字 · 拒绝多音字 · 拒绝玄学堆砌 · 追求常用字的高级组合 · 诗意留白 · 民国文人气质 |
| **商业模式** | Freemium — 基础起名/打分免费，深度解读/无限生成/报告导出付费 |
| **平台** | Web（响应式，Mobile-first） |

## Phase 1 · 策略 & 架构

### 功能优先级

| 优先级 | 功能 | 说明 |
|---|---|---|
| 🔴 核心 | **起名生成** | 输入姓氏+性别+偏好+自由描述 → AI 生成 8 个候选名 |
| 🔴 核心 | **名字打分** | 输入完整名字 → 100 分制综合评分 + 5 维解读 |
| 🔴 核心 | **5 维打分展示** | 雷达图 + 逐维解读（见下方维度设计） |
| 🟡 重要 | **偏好设定** | 风格选择（大雅/大俗/不限）、排除字、辈分字、自由描述 |
| 🟡 重要 | **收藏 & 对比** | 收藏名字，多名字横向对比 |
| 🟡 重要 | **重名查询** | 展示大致重名率 |
| 🟢 锦上添花 | **PDF 报告导出** | 付费功能 |
| 🟢 锦上添花 | **家族取名记录** | 多宝宝取名历史 |
| 🟢 锦上添花 | **社区/排行榜** | 热门名字趋势 |

### 5 维打分体系 v2

| 维度 | 分值 | 核心标准 |
|---|---|---|
| **🔊 音韵搭配** | 20 | 声调起伏、声韵搭配、朗读节奏感 |
| **🔤 字形美感** | 20 | 笔画均衡、结构搭配协调、书写美感 |
| **🏔️ 大雅大俗** | 20 | **双路径评分**（见下方详解） |
| **🖋️ 民国风骨** | 20 | 温润克制、有典不造作、穿越时间仍优雅 |
| **📊 实用性** | 20 | 无生僻字/多音字/谐音陷阱、重名率 |

#### 「大雅大俗」双路径评分

AI 先判断名字属于哪条路线，再按对应标准打分：

| 路线 | 高分标准 | 示例 | 低分反例 |
|---|---|---|---|
| **大雅**：禅意·水墨·文人 | 想象空间大、意象含蓄 | 半亩、见山、修远、听泉 | 美丽、聪慧（太直白） |
| **大俗**：泥土·原野·生命力 | 画面质朴、生命力蓬勃 | 田野、麦田、大川、春生 | 招财、旺福（俗无力量） |

### 信息架构

```
首页（Landing）
├── 起名入口 → 输入页（姓氏/性别/风格/辈分字/自由描述）
│                 └── 结果列表（8 个名字卡片）
│                       └── 名字详情（5 维雷达图，侧面板展开）
├── 打分入口 → 输入页（完整姓名）
│                 └── 打分结果（总分 + 雷达图 + 逐维解读）
├── 我的收藏 / 对比（需登录）
└── 付费解锁页
```

## Phase 2 · 交互 & 流程

### 起名流程（Happy Path）

```
输入页（单页）──→ AI 生成（3-5秒，水墨动画）──→ 结果列表 ──→ 名字详情
```

**输入页整合设计：**
- 姓氏（必填）+ 性别（必选）+ 风格偏好（卡片选择器）+ 辈分字（选填）
- **✨ 自由输入框**（选填，≤200 字）：用户用自己的话描述偏好，直接传给 LLM
- Placeholder 随机轮播引导示例，如"希望有山水意境，清新温暖的感觉"

| 交互决策 | 方案 | 依据 |
|---|---|---|
| 单页 vs 多步 | 单页（输入项 ≤5） | 字段少时单页转化率更高 |
| 生成数量 | 免费 8 个/批 | Miller's Law：7±2 信息块 |
| 详情展开 | 侧滑面板，不跳页 | 保持列表上下文 |
| 等待态 | 水墨动画 + 起名趣闻轮播 | 将等待转化为仪式感 |

### 打分流程（Happy Path）

```
输入名字 ──→ AI 分析（2-3秒）──→ 打分结果（雷达图 + 路线标签 + 逐维解读）
```

极简单输入框，门槛降到最低。

### 异常路径处理

| 场景 | 方案 |
|---|---|
| 复姓 / 罕见姓 | 自动识别复姓库，非标姓给确认提示 |
| AI 超时（>8s） | "好名字值得等" + 自动重试 1 次，保留输入项 |
| 全不满意 | 「换一批」免费 3 次/天；第 2 次换批收集偏好反馈微调 Prompt |
| 打分不认同 | 底部 👍👎 反馈 + 一句话反馈框 |
| 免费次数用完 | 结果模糊预览 + 解锁按钮（非硬阻断弹窗） |

### 付费分割线

| 免费 | 付费解锁 |
|---|---|
| ✅ 8 个名字推荐 | 🔒 无限换一批 |
| ✅ 总分 + 路线标签 + 一句话解读 | 🔒 完整 5 维详细解读 |
| ✅ 简要雷达图 | 🔒 PDF 报告导出 |
| ✅ 收藏+对比（3 个） | 🔒 无限收藏+对比 |

## Prompt 设计

### 起名生成 System Prompt

```
角色：中文起名大师，精通古典文学与民国文人命名风格。

硬性约束：
1. 只用常用字（3500 字范围），不使用生僻字
2. 不使用多音字（乐/行/重/朝等）
3. 回避近 10 年烂大街名字（梓涵/子涵/浩然/欣怡/雨桐/可馨/
   诗涵/梓萱/一诺/奕辰/沐辰/若汐/语桐等）
4. 回避谐音陷阱
5. 2 字名或 3 字名（含姓）

审美标准：
- 大雅路线：意象含蓄、想象空间大、禅意水墨
  参考：林徽因/沈从文/杨绛 | 示例：见山/修远/半亩/听泉
- 大俗路线：画面质朴、生命力蓬勃、接地气有力量
  参考：沈从文湘西/汪曾祺乡土 | 示例：田野/麦田/大川/春生

推荐策略：
- 选"大雅"：6 雅 2 俗 | 选"大俗"：2 雅 6 俗 | 选"不限"：4-4
- 分数有梯度，风格有变化

输出：严格 JSON（8 个名字，含 full_name/route/score/one_liner/5 维分项）
```

### 打分 System Prompt

```
5 维评分（每项 0-20，总分 100）：

1. 音韵搭配：声调组合、声母搭配、朗读节奏
   锚点：18-20=极舒适 | 14-17=中等 | <14=明显拗口

2. 字形美感：结构搭配变化、笔画均衡、毛笔书写美感
   锚点：18-20=有变化且均衡 | <14=单调或失衡

3. 大雅大俗：先判路线，再按路线标准打分
   大雅→留白越大越高 | 大俗→生命力越蓬勃越高
   又直白又无力量 ≤10 分

4. 民国风骨：常用字非常见组合、温润克制、有典不造作、时代穿透力
   过度网感/流行/华丽→减分

5. 实用性：含生僻字 ≤10 | 含多音字 ≤12 | 检查谐音+重名率

一致性规则：temperature=0 | 分数拉开梯度 | 禁止泛泛而谈

输出：严格 JSON（total_score/route/route_reason/5 维分项含 analysis/overall_comment）
```

## Phase 3 · 视觉 & 布局

### 视觉方向：「新文人」

| 维度 | 方向 | 拒绝 |
|---|---|---|
| 气质 | 克制、留白、温润 | 玄学金光、赛博霓虹 |
| 色彩 | 低饱和中国色 | 高饱和大红大金 |
| 字体 | 标题宋体、正文黑体 | 全站毛笔楷书 |
| 留白 | 大面积，低密度 | 信息堆砌满屏 |
| 动效 | 缓慢流动、水墨感 | 弹跳闪烁粒子 |

### 色彩体系

| 色名 | 色值 | 用途 |
|---|---|---|
| 黛色 | `#4A5568` | 标题 / 强调文字 |
| 月白 | `#F7F3EE` | 页面背景 |
| 烟灰 | `#A0AEC0` | 辅助文字 |
| 净白 | `#FFFFFF` | 卡片底色 |
| 藕荷 | `#C4A882` | CTA 按钮 / 高分亮点 |
| 赭石 | `#B7856A` | 大俗标签（暖·质朴） |
| 竹青 | `#7BA08A` | 大雅标签（冷·禅意） |

### 核心页面布局

#### 首页
- 极简，居中两个场景化入口：「给宝宝取名」「看看名字好不好」
- 底部散落示例名字（低透明度水墨色），暗示调性
- 无复杂导航栏

#### 起名输入页
- 窄栏居中（max-width ~600px），保持"宣纸"感
- 风格偏好用卡片选择器（附示例名字）
- 自由输入框用下划线式，呼应"宣纸上写字"

#### 结果列表页
- 名字卡片：大字号名字 + 分数 + 路线色标签 + 一句话解读
- 卡片间大间距，独立呼吸
- 点击展开侧面板查看详情

#### 名字详情面板
- 5 维雷达图（低饱和填充、细线条）
- 逐维解读向下展开
- 底部 CTA：收藏 + 解锁完整报告

### 关键组件

| 组件 | 方向 |
|---|---|
| 名字卡片 | 大留白 + 宋体大字 + 分数徽章 + 路线色带 |
| 雷达图 | 低饱和填充、细线条、无网格底纹 |
| CTA 按钮 | 藕荷色、圆角、hover 微发光 |
| Loading | 水墨晕染 / 毛笔书写动画 |
| 路线标签 | 药丸形状，竹青 🏔 / 赭石 🌾 |
| 输入框 | 下划线式，减少视觉重量 |

## 风险总览 & 对策

| # | 风险 | 对策 |
|---|---|---|
| 1 | AI 生成名字"惊喜感"不足 | Prompt 加负面清单 + 民国名字 few-shot |
| 2 | 同一名字多次打分不一致 | temperature=0 + 锁定评分锚点 |
| 3 | 免费用户 API 成本过高 | 每天 3 次起名 + 5 次打分；付费解锁 |
| 4 | 「大雅大俗」路线判断错误 | Chain-of-Thought + 向用户展示路线标签可纠错 |
| 5 | 自由输入引入不可控内容 | 防护指令 + 字数截断 + 敏感词过滤 |
| 6 | 「轻五行」流失付费用户 | 五行降级为"参考维度"，不完全砍掉 |
| 7 | 用户敏感信息合规 | 出生日期设为可选，明示数据用途 |
| 8 | 新文人风格可能显"寡淡" | 通过微动效注入灵动感，静态克制+动态灵动 |
